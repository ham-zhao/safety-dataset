# ML 项目方法论（精简版）

> 直接粘贴到项目 CLAUDE.md 中作为 prompt 前缀。<500 字。

---

## 元原则

- **中性化**：所有规则与具体项目/工具解耦，可跨项目复用
- **三阶段**：每个环节都有事前（防方向错）、事中（防偏离）、事后（防遗漏）检查
- **输出必有结论**：每个图表/数字后必须有判断（正常/异常/需关注）+ 基准对比 + 影响说明

## 设计阶段

1. **五要素先行**：每个模块先定义 输入/输出/过程/目标/度量（含基准值），再写代码
2. **MECE 检查**：模块不重叠（ME）、整体不遗漏（CE）
3. **六维度对齐**：问题→方法、数据→任务、模型→数据、过程→严谨性、指标→目标、解释→现实
4. **假设显式化**：偏离论文标准的假设必须声明理由和影响

## 数据阶段

5. **数据源匹配任务**：脏数据测清洗、干净数据训模型、独立数据做评估
6. **样本量倒推**：从末端需求倒推输入量，每步输出 ≥50 条
7. **分布对齐**：训练/推理数据在干扰变量（长度等）上同分布，消除 Covariate Shift
8. **数据血缘**：Agent 用 AST 自动构建依赖图，修改上游时提示下游影响

## 执行阶段

9. **节点标准化**：每个 Pipeline 节点有卡片（输入/输出/基准/预期/验收/失败处理）
10. **漏斗指标**：多步 Pipeline 有漏斗表，从大步骤细拆到子步骤
11. **配置驱动**：所有数量参数在配置文件中，代码不硬编码
12. **增量验证**：写一步跑一步验一步，步骤 N 未通过禁止开始 N+1

## 分析阶段

13. **层次化评估**：原子指标→组件发现→系统结论→方法论结论，逐层聚合
14. **统计差异先调查**：偏差 >2x 禁止直接下结论，先抽查案例 + 技术核验
15. **ML 健康检查**：训练后必做——分数分布/正负分离度/黄金样本/基线对比/数据版本
16. **独立外部评估**：评估器与 pipeline 不重叠、不同架构、不参与决策

## 交付阶段

17. **口径核验**：引用论文时列出条件差异和影响评估
18. **指标 SSOT**：口径统一定义一次，全文引用不重复。百分比必须写分子/分母
19. **金字塔结论**：结论先行，支撑论点 ≤3 个

## Agent 行为

20. **指令消歧**：模糊指令主动列出可能解释，请用户确认
21. **自检机制**：每步后检查输出文件、核心指标、是否遗漏子需求
22. **跨模块一致性**：术语、口径、数据格式在上下游保持一致
